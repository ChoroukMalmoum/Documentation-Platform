CustomYoloModel

class CustomYoloModel(in_channels=3, architecture=None, img_size=448, activation='LeakyReLU', **kwargs)

A class that represents a custom YOLO (You Only Look Once) model.

This class is a subclass of the PyTorch nn.Module class. An instance of this class represents a YOLO model, which is a type of Convolutional Neural Network (CNN) used for object detection tasks.

Parameters:

  • in_channels (int, optional) – Number of input channels. Default is 3.

  • architecture (list, optional) – The architecture of the CNN. Each element in the list represents a layer in the network. Default is an empty list.

  • img_size (int, optional) – The size of the input images. Default is 448.

  • activation (str, optional) – The type of activation function to use. Default is ‘LeakyReLU’.

  • kwargs (dict, optional) – Arbitrary keyword arguments for the fully connected layers.

Variables:

  • architecture (list) – The architecture of the CNN. Each element in the list represents a layer in the network.

  • in_channels (int) – Number of input channels.

  • out_channels (int) – Number of output channels.

  • stride (int) – The stride of the convolution.

  • split_size (int) – The size of the split.

  • img_size (int) – The size of the input images.

  • activation (str) – The type of activation function to use.

  • darknet (nn.Module) – The Darknet part of the YOLO model, which is a type of CNN.

  • fcs (nn.Module) – The fully connected layers of the model.

  • add_conv_layer: This method allows users to add a convolutional layer to the architecture with the specified parameters (kernel size, number of output channnels, stride, padding).

  • add_maxpool_layer: This method allows users to add a max pooling layer to the architecture.

  • add_dense_block: This method adds a dense block to the architecture. A dense block consists of two convolutional layers that are repeated a certain number of times.

  • remove_last_layer: This method removes the last layer from the architecture.

  • remove_last_n_layers: This method removes the last n layers from the architecture.

  • remove_all_layers: This method removes all layers from the architecture.

  • modify_layer: This method modifies the specified layer in the architecture.

  • add_conv_layer_before_maxpool: This method adds a convolutional layer before a max pooling layer in the architecture.

  • validate_architecture: This method validates the model architecture to ensure it is potentially effective.

Note

Once the chosen architecture is validated, it is possible to train the model using our train_Custom_model function. This function takes care of the entire training process, including loading the data, setting up the training loop, and saving the trained model.
